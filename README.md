# DataWitness
### Trust-Aware Conversational Data Analyst

> Built for **NatWest Code for Purpose -- India 2026** | Theme 1: Talk to Data

---

## Overview

DataWitness lets anyone ask natural language questions about a dataset and instantly receive clear, verified, and fully traceable answers -- with no SQL knowledge required.

Unlike generic chatbots, DataWitness never lets the AI do the math. The LLM translates your question into a deterministic Pandas query, which is executed on the actual data. Every answer shows the exact query used, the metric definitions applied, and the source rows it was derived from -- eliminating the black-box problem entirely.

**Who it's for:** Business analysts, managers, and non-technical users who need to trust the data answers they act on.

---

## Features

- **Natural Language Querying** -- Ask questions in plain English about any structured CSV dataset
- **Intent-Routed Reasoning** -- Questions are classified (change / compare / breakdown / counterfactual / summarise) and routed to specialised Pandas engines for instant deterministic answers -- no LLM math
- **Audit Trail Split-Screen** -- Left panel shows the narrative answer and chart; right panel shows the exact Pandas code executed, confidence score, metric definitions, and source rows
- **Trust Layer** -- Every answer includes a confidence score (0-100), metric definitions from `metrics.yaml`, and source row references
- **Self-Correcting Execution** -- If generated Pandas code fails, the traceback is fed back to the LLM to fix it (up to 2 retries)
- **Dynamic Schema Injection** -- Only relevant columns and metrics are injected into the LLM prompt using fuzzy matching, reducing noise and token usage
- **Deterministic Time Parsing** -- Relative time phrases ("last month", "year to date") are resolved by Python, not the LLM, using exact timedelta logic
- **ELI5 Toggle** -- One-click switch between Analyst Mode (precise numbers) and Plain English Mode (jargon-free)
- **Auto Visualisation** -- Charts are auto-selected based on intent: waterfall for change, grouped bar for compare, breakdown bar with top contributor highlighted
- **Knowledge Graph** -- Interactive reasoning graph (via pyvis + vis-network) shows metric nodes, period nodes, and contributor nodes for change and compare queries
- **Smart Insights** -- On dataset load, 5 patterns are auto-discovered: trends, top entities, anomalies (2 sigma), concentration, and correlations
- **Dataset Profiler** -- Upload any CSV and the schema is auto-detected (metrics, dimensions, time columns) without needing a hand-written `metrics.yaml`
- **CSV Upload** -- Works with any structured CSV, not just the sample dataset
- **Breakdown Engine** -- Decomposes metrics by dimension with outlier detection, percentage contribution, cumulative percentages, and a 4-sentence analyst narrative
- **Follow-Up Suggestions** -- After each query, 3 contextual follow-up questions are generated by the LLM
- **Query History** -- Last 5 queries shown with intent badges and confidence ratings

## Install and Run

### Prerequisites
- Python 3.10 or higher
- Node.js 18 or higher
- A free Groq API key -- get one at [console.groq.com](https://console.groq.com)

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/Pikachu-345/data-witness.git
cd data-witness

# 2. Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Set up environment variables
copy .env.example .env       # Windows
# cp .env.example .env       # Mac/Linux
# Then open .env and add your GROQ_API_KEY

# 5. Start the backend (FastAPI on port 8000)
uvicorn api.main:app --reload --port 8000

# 6. In a second terminal, start the frontend
cd frontend
npm install
npm run dev
```

Open your browser at **http://localhost:5173**

Alternatively, on Windows you can run `start.bat` which launches both servers automatically.

### Using the sample dataset

The Superstore Sales CSV is included at `assets/superstore.csv`. On first load the app offers a "Use Sample Data" button that loads it automatically.

## Usage Examples

**Understand what changed:**
> "Why did profit change between 2016 and 2017?"

Routes to the change detection engine. Returns period-over-period comparison, top contributors ranked by impact, waterfall chart, and contributor bar chart.

**Compare:**
> "Compare sales in the West vs East region"

Routes to the comparator engine. Returns side-by-side values, winner determination, percentage difference, and a grouped bar chart.

**Breakdown:**
> "What makes up total sales by category?"

Routes to the breakdown engine. Returns ranked components with percentage contribution, cumulative percentages, outlier detection, a 4-sentence analyst narrative, and a bar chart with the top contributor highlighted.

**Summarise:**
> "Give me a summary of overall business performance"

Routes to the summariser. Returns multi-metric statistical summary with trend direction, anomaly counts, and top/bottom performers.

**Counterfactual:**
> "What if the South region didn't lose profit?"

Routes to the counterfactual engine. Compares the actual outcome vs a hypothetical scenario where a specific entity's change is removed.

Each answer returns:
- A plain-English narrative (or ELI5 version)
- An auto-generated chart matched to the query intent
- The exact Pandas query used (for general queries)
- Confidence score + metric definitions
- Source rows from the raw data
- 3 LLM-generated follow-up questions

## Architecture

```
User Question (React Frontend)
       |
       v
  FastAPI Backend (api/main.py)
       |
       v
  Intent Classifier (Groq LLM)
  Classifies: change / compare / breakdown / counterfactual / summarise / general
       |
       +---> change    --> change_detector.py (pure Pandas, no LLM)
       +---> compare   --> comparator.py (pure Pandas, no LLM)
       +---> breakdown --> breakdown.py (pure Pandas + outlier detection)
       +---> counterfactual --> counterfactual.py (pure Pandas)
       +---> summarise --> summarizer.py (pure Pandas)
       +---> general   --> query_engine.py:
       |                     NL --> Groq LLM --> Pandas code string
       |                     --> Safety checker (blocks import/exec/os/sys)
       |                     --> Python exec() on real DataFrame
       |                     --> Self-correction loop (up to 2 retries)
       |
       v
  trust.py (confidence score + metric detection + source rows)
       |
       v
  Groq LLM generates narrative from verified result (never sees raw data)
       |
       v
  React Frontend: Answer + Charts + Knowledge Graph + Audit Trail
```

For a more detailed architecture breakdown, see [docs/architecture.md](docs/architecture.md).

## Limitations

- Works best with structured, tabular CSV data with consistent column names
- Very complex multi-step analytical questions may require rephrasing
- The confidence score is heuristic-based, not statistically derived
- Large datasets (>500k rows) may have slower query times on free-tier hardware
- The self-correcting loop adds latency when retries are needed (up to 2 extra LLM calls)
- Knowledge graphs are only generated for change and compare query intents
